Travel Tools 系統技術規格書

v2.0 - 行李打包與離線憑證升級版

系統概述

Travel Tools (旅遊工具箱) 是一款專為自助旅行者設計的單頁應用程式（Single Page Application, SPA）。 本系統整合了匯率換算、時區時鐘、記帳、翻譯、待辦清單、多行程管理及離線憑證夾等必備工具。

為了解決海外旅行時常遇到的「高流量漫遊限制」或「在機場與地下交通缺乏網路」等痛點，本系統採用 100% 離線可用 的設計哲學。 所有行程、財務、照片與備忘錄數據均僅存儲於用戶端的瀏覽器本地中，保障隱私與離線可用性。

技術架構

系統採用輕量級的無伺服器（Serverless）純前端架構，無須資料庫或後端伺服器，直接透過 CDN 引入成熟的庫：

元件 技術 / 來源 功能描述
UI 基礎 W3.CSS (CDN) / Vanilla CSS3 輕量響應式框架，配合 CSS 自訂屬性（Variables）支援黑暗模式（Dark Mode）。
核心框架 Vue.js v3 (CDN Global) 資料驅動 UI，使用 Options API 管理反應式狀態。
拖曳元件 SortableJS (CDN) 用於待辦清單 (Todo) 行李品項的無縫觸控與滑鼠拖曳排序。
照片編輯器 Cropper.js v1.6.2 (CDN) 支援拍照/上傳後的「自由裁切」與「旋轉擺正」，極大化節省儲存空間。
資料快取 Service Worker (sw.js) 實作 PWA (Progressive Web App) 快取策略，快取 HTML/CSS/JS 以及 CDN 靜態資源。

功能模組規格

1. 計算器與換算器

整合四則運算、開根號 (n√x)、次方 (x^n) 及百分比。 幣值兌換模組使用 ExchangeRate-API 進行即時匯率轉換，並在載入成功後將匯率存入 localStorage 作為離線降級快取。 貨幣選擇具備記憶功能，會記住使用者最後一次選擇的原始與目標幣別。

2. 世界時鐘與時區換算

支援動態計算兩地時差。 下方設有 **8 大熱門城市世界時鐘網格**，系統會以 Taipei 時間作為基準（不依賴系統本地時間以防止時差計算出錯），每秒計算一次時間，並配合經緯度動態計算日出 (☀️) 與日落 (🌙) 狀態。

3. 待辦備忘 (行李清單)

支援 Shift-Range 批量勾選與 Ctrl 點選。待辦項目支援長按/拖曳自訂排序。 一鍵導入 **🎒 行李打包範本** 邏輯會將 8 項核心旅行物品（如護照、外幣、插頭等）以「追加（不覆蓋現有品項）」的方式推入 Vue `todos` 狀態中並自動存檔。

4. 旅遊記帳與甜甜圈圖表

支援記錄消費備註、金額及類別。外幣消費會依當前匯率自動換算折合為台幣。 在總額下方使用 HTML5 原生 Canvas 繪製 Donut Chart，會動態統計「餐飲、購物、交通、住宿、門票、紀念品、其他」等類別並依金額由高到低自動著色與排序。 支援在列表中點選「鉛筆」圖示彈出編輯視窗，修改後即時重繪圖表。

5. 離線重要憑證相簿 (Travel Wallet)

特設折疊抽屜式卡片 UI，點選標題可展開/收合。限制最大照片上傳上限為 4 張。 選取圖片時（可直接用「相機拍攝」或「檔案選擇器」讀取 Google Drive 檔案），會觸發 **Cropper 編輯視窗**，在毛玻璃彈窗中使用者可自由拖曳裁切或左右轉 90° 擺正圖片。 確定後，程式會先進行裁切，隨後利用 Canvas 將圖片等比縮減至最大邊 1000px，並以 image/jpeg 0.7 品質壓縮，將單張 Base64 體積壓在 100KB 左右以防止 LocalStorage 空間不足。

6. Gemini AI 雙通道翻譯與常用語

支援 API Key 設定（具備隱私遮罩），密鑰直接保存在本地，以 Gemini 2.0 API 驅動翻譯。 若無設定金鑰，系統會無縫 fallback 到免費翻譯 API，以保障使用者在戶外的翻譯通暢。 「我的常用語」區塊可讓使用者新增高頻會話並保存，並整合了 Web Speech API 語音合成（TTS）點擊即可發音。

本地儲存結構

整個 App 為了支援多行程，採用了一個名為 Version 2.0 的資料結構。 在 LocalStorage 中，行程主體以 travel_trips 為 Key 進行儲存： 

{
  "version": "2.0",
  "activeTripId": "trip_1780400000"000,
  "trips": [
    {
      "id": "trip_1780400000",
      "name": "東京賞櫻之旅 🌸",
      "departureDate": "2026-04-01T08:00",
      "itineraryText": "<div>Day 1: 抵達新宿...</div>",
      "todos": [
        { "id": "todo_1", "text": "護照", "important": true, "completed": false }
      ],
      "expenses": [
        { "remark": "拉麵", "category": "餐飲", "amount": 1200, "currency": "JPY" }
      ],
      "walletPhotos": [
        { "id": "photo_1", "name": "登機證", "base64": "data:image/jpeg;base64,/9j/4AAQSk..." }
      ]
    }
  ]
}
⚠️ 注意事項 由於 walletPhotos 採用 Base64 格式嵌入行程陣列中，當用戶執行「匯出備份」時，所有照片會隨同 JSON 自動打包下載；匯入時也會一同載入。請提醒使用者避免匯入非經前端壓縮的超大 JSON 檔案。

PWA 與離線快取

透過 manifest.json 將網頁封裝為可安裝之 Progressive Web App (PWA)。 核心快取策略定義在 sw.js 中：

  • ASSETS_TO_CACHE：預先快取 index.html、manifest.json 之外，亦包含 Vue、SortableJS 以及 CropperJS 的外部 CDN 樣式與腳本。
  • 動態 CDN 快取 (Cache-First)：凡是來自 cdnjs.cloudflare.com、unpkg.com、jsdelivr.net 等 CDN 的資源要求，一律優先使用快取，無快取時才連網下載並動態寫入快取，以確保 100% 離線可用。
  • 匯率 API 快取 (Network-First, Fallback-to-Cache)：匯率請求優先使用網路以獲取最新報價，若處於離線狀態則自動 fallback 讀取本地上一次儲存的匯率快取。

維護與擴充指南

本地開發

由於 Service Worker 安全限制，無法透過雙擊 HTML 運行。請在專案目錄下啟動輕量伺服器：

python -m http.server 8000

隨後即可在瀏覽器中開啟 http://localhost:8000/index.html 進行測試。

版本升級

當您修改了 index.html 或專案資源時，必須手動更新 sw.js 中的 CACHE_NAME 版本號（例如從 v27 升級至 v28）。 這樣使用者瀏覽器在連網時，Service Worker 才會自動下載最新檔案、觸發啟用事件並清空舊快取，避免使用者因快取鎖死而停留在舊版畫面。
